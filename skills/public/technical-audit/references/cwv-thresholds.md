# Core Web Vitals 阈值参考

## 三大核心指标

### 1. LCP (Largest Contentful Paint)

**定义：** 最大内容绘制时间，衡量页面加载速度

| 评级 | 阈值 | 说明 |
|------|------|------|
| ✅ 良好 | < 2.5 秒 | 用户感觉加载很快 |
| ⚠️ 需改进 | 2.5-4.0 秒 | 用户可能感到轻微延迟 |
| ❌ 差 | > 4.0 秒 | 用户体验明显受损 |

**优化建议：**
- 优化服务器响应时间（TTFB）
- 使用 CDN 加速静态资源
- 压缩和优化图片
- 预加载关键资源
- 移除阻塞渲染的 CSS/JS

---

### 2. CLS (Cumulative Layout Shift)

**定义：** 累积布局偏移，衡量页面视觉稳定性

| 评级 | 阈值 | 说明 |
|------|------|------|
| ✅ 良好 | < 0.1 | 几乎无布局跳动 |
| ⚠️ 需改进 | 0.1-0.25 | 偶尔有跳动 |
| ❌ 差 | > 0.25 | 频繁跳动，影响使用 |

**优化建议：**
- 为图片和视频设置明确的宽高
- 避免在现有内容上方插入新内容
- 使用 aspect-ratio CSS 属性
- 预留给广告和嵌入内容空间
- 使用字体加载策略避免 FOIT

---

### 3. INP (Interaction to Next Paint)

**定义：** 交互到下次绘制，衡量页面响应性

| 评级 | 阈值 | 说明 |
|------|------|------|
| ✅ 良好 | < 200 毫秒 | 响应流畅 |
| ⚠️ 需改进 | 200-500 毫秒 | 偶尔感觉卡顿 |
| ❌ 差 | > 500 毫秒 | 明显延迟 |

**优化建议：**
- 减少 JavaScript 执行时间
- 使用 Web Workers 处理复杂计算
- 优化事件监听器
- 避免长任务（>50ms）
- 使用 requestIdleCallback 处理非关键任务

## 移动端 vs 桌面端

**注意：** Google 主要使用**移动端**数据进行排名

| 设备 | 网络条件 | CPU 限制 |
|------|----------|----------|
| 移动端 | 4G | 4 倍 CPU 降速 |
| 桌面端 | 宽带 | 无降速 |

**建议：** 优先优化移动端指标

---

## 目标设定

| 指标 | 第 75 百分位目标 | 监测频率 |
|------|------------------|----------|
| LCP | < 2.5s | 每周 |
| CLS | < 0.1 | 每周 |
| INP | < 200ms | 每周 |

**注意：** 使用第 75 百分位（而非平均值）来评估，确保大多数用户体验良好
